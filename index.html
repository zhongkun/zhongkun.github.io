流畅的Android 模拟器:Genymotion

六 13 七月 2013

tags: ToolsJava

Genymotion is a complete set of tools that provides a virtual environment for Android. It can be very useful for developers, testers, salesman or even gamers. It is available for most operating systems: Windows, Linux and MacOS X. It is as simple to install as powerful to use: Follow the simple install process, choose one of our Android virtual devices, start it, and enjoy it

Android模拟器卡顿，是经常被iOS开发者经常吐槽的一点. Android的模拟器都是基于ARM的体系结构(arm-eabi),因而模拟器不是直接运行在x86上，而是模拟ARM,iOS 和WP7的模拟器的结构都是x86的,另外iOS的模拟器中运行的App也是编译为x86的。 Android 模拟器用的是 QEMU 的全系统模式(full system)，也就说它模拟时需要启动整个 GUEST 系统，初始化各种模拟设备。相反的，iOS 和 WP7 的模拟器只是 user-mode 的模拟。 在 Android 模拟器之中还需要跑一个 Dalvik VM，用以执行 Android 应用的 bytecodes.

So

目前有三个解决方案

1) 2011年的解决方案: 利用开源的 Android-x86 项目，运行在VirualBox里. 可以参考这篇blog 用Android-X86和VirtualBox打造高性能Android开发环境

2) 2012年,Intel 推出了官方Intel Atom x86 System Image. 支持x86的模拟器，这将允许Android模拟器能够以原始速度（真机运行速度）运行在使用Intel x86处理器的电脑中, 可以去Intel 官网下载( http://software.intel.com/en-us/articles/intel-hardware-accelerated-execution-manager/) ,也可以在SDK Manager 里直接下载.
