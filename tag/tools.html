流畅的Android 模拟器:Genymotion

六 13 七月 2013

tags: ToolsJava

Android模拟器卡顿，是经常被iOS开发者经常吐槽的一点. Android的模拟器都是基于ARM的体系结构(arm-eabi),因而模拟器不是直接运行在x86上，而是模拟ARM,iOS 和WP7的模拟器的结构都是x86的,另外iOS的模拟器中运行的App也是编译为x86的。

Android 模拟器用的是 QEMU 的全系统模式(full system)，也就说它模拟时需要启动整个 GUEST 系统，初始化各种模拟设备。 相反的，iOS 和 WP7 的模拟器只是 user-mode 的模拟。 在 Android 模拟器之中还需要跑一个 Dalvik VM，用以执行 Android 应用的 bytecodes.

So

目前有三个解决方案

1) Android-x86 + VirtualBox

2011年的解决方案: 利用开源的 Android-x86 项目，运行在VirualBox里. 可以参考这篇blog 用Android-X86和VirtualBox打造高性能Android开发环境

2) HAXM

2012年,Intel 推出了官方Intel Atom x86 System Image. 支持x86的模拟器，这将允许Android模拟器能够以原始速度（真机运行速度）运行在使用Intel x86处理器的电脑中。

先下载Intel® Hardware Accelerated Execution Manager 可以去Intel 官网下载( http://software.intel.com/en-us/articles/intel-hardware-accelerated-execution-manager/) ,也可以在SDK Manager Extra 勾选Intel x86 Emulator Accelerator(HAXM) 直接下载. (目前最新版是R3),然后下载对应系统版本的Intel x86 Atom System Image.

可以看下Intel 官方文档 Installation Instructions for Intel® Hardware Accelerated Execution Manager - Microsoft Windows*

当使用Android SDK提供的"emulator-x86"二进制文件时， Intel® x86 Android模拟器镜像会自动执行， 这个时候，如果Intel® HAXM已经成功安装并执行了，这个镜像将会充分使用Intel® VT 虚拟化技术。

关于HAXM更多的信息，可以看这里: 英特尔硬件加速管理器HAXM安装使用指南

3)GenyMotion

Genymotion is a complete set of tools that provides a virtual environment for Android. It can be very useful for developers, testers, salesman or even gamers. It is available for most operating systems: Windows, Linux and MacOS X. It is as simple to install as powerful to use: Follow the simple install process, choose one of our Android virtual devices, start it, and enjoy it

项目地址:http://www.genymotion.com/

其实上面两个方案已经能解决模拟器卡顿的问题了，但我们需要一个更加强大的模拟器来提高效率，GenyMotion 也是基于VirualBox 的方案，对比系统的模拟器不仅速度快，而且提供了很多更强大的功能。 看下他的Feature:

  • 可以直接在Virual Devices 里登录Google 账户,在Play 直接下软件，开gtalk,gmail等。

  • 能够打全局的root 权限

  • 可配置硬件加速

  • 可选择超大屏幕(Android TV)

  • 提供了Intellij IDEA 插件，可以在IDEA里方便的开启调试模拟器

  • 可以设置Wifi ,3G 网络， 动态设置GPS, 甚至能调用喜糖蓝牙

  • 可以调节电量大小，陀螺仪，屏幕亮度，温度传感器，加速度传感器...

  • 有一个强大的Configuration shell ，做自动化测试会更加方便

....

Genymotion 基于VirualBox 4.1+,目前Mac 的版本还是1.0.0，已经有13个Virtual device 可以安装，需要安装With Google Apss 的devcie才会自带Google 应用包。目前模拟的设备有Nexus 7,Nexus S,Nexus One ,Galaxy Nexus, Table. 最后来一张用模拟器下app截图
